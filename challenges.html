About CRS Challenges

The CRS (Code Review Security) challenges are designed to test AI models' capabilities in identifying security vulnerabilities and generating patches across different programming languages and challenge types. Each challenge focuses on specific security issues commonly found in real-world applications.

Challenge Types Overview

Full-Mode

Full Challenges

Description: Complete vulnerability scenarios requiring comprehensive analysis.

Scope: End-to-end security assessment

Focus: Real-world attack vectors and exploitation

Difficulty: High to Very High

Delta-Mode

Delta Challenges

Description: Incremental security patches and vulnerability fixes.

Scope: Specific vulnerability remediation

Focus: Code review and patch generation

Difficulty: Medium to High

Scoring System

POV (Proof of Vulnerability): 2 points each - Awarded for successfully identifying and demonstrating a security vulnerability.

Patch: 6 points each - Awarded for generating effective security patches that fix the identified vulnerabilities.

Total Score = (POVs × 2) + (Patches × 6)
